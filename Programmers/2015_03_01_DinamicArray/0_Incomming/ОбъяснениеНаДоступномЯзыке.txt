Задача - хранить кучу объектов типа int. Для этого надо создать массив int*
Любой массив имеет конечную длину. Обозначим её 256.
Если нам надо хранить больше объектов, то мы должны создать ещё массивы.
В результате, чтобы хранить X объектов, нам надо хранить X/256 указателей на массив int[256]

Задача - хранить кучу объектов типа int*. Для этого надо создать массив int**
Любой массив имеет конечную длину. Обозначим её 256.
В результате, чтобы хранить X/256 указателей, нам надо хранить X/256/256 указателей на указатели на массив int[256]

Задача - хранить кучу объектов типа int**. Для этого надо создать массив int***
Любой массив имеет конечную длину. Обозначим её 256.
В результате, чтобы хранить X/256/256 указателей, нам надо хранить X/256/256/256 указателей на указатели на указатели на массив int[256]

Задача - хранить кучу объектов типа int***. Для этого надо создать массив int****
Любой массив имеет конечную длину. Обозначим её 256.
В результате, чтобы хранить X/256/256/256 указателей, нам надо хранить X/256/256/256/256 указателей на указатели на указатели на массив int[256]

Положим, Х не превышает 256^5 = 2^40 = 1000 Гига элементов, что не такая уж маленькая величина.


Резульат:
Переменная int может хранить 1 значение
Переменная int* -    256  значений
Переменная int** -    64К значений
Переменная int*** -   16M значений
Переменная int**** -   4G значений
Переменная int***** -  1T значений
